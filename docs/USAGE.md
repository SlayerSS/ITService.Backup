# ITBackup — инструкция

Простая **ручная** программа: вы сами решаете, когда сделать бэкап, и нажимаете **одну кнопку**.  
Сделана в первую очередь для **1С на Microsoft SQL Server**.

## Кому подходит

- Сервер или ПК с **1С** и **SQL Server**
- Нужна копия **на флешку**, диск или в папку
- Не нужен сложный софт с расписанием — достаточно «сделал сейчас»

## Перед первым запуском

1. Скачайте с [Releases](https://github.com/SlayerSS/ITService.Backup/releases/latest):
   - **ITBackup-net8.exe** — если на сервере есть [.NET 8 Desktop](https://dotnet.microsoft.com/download/dotnet/8.0);
   - **ITBackup-standalone.exe** — если .NET нет.
2. Запускайте **от имени администратора** (нужно для SQL и копирования занятых файлов).
3. Подготовьте **USB-флешку** или укажите другое место в настройках.

## Главное окно

- Вверху — подсказки по безопасности и работе с 1С.
- Блок **носителя**: статус USB или диска, свободное место; если флешек несколько — переключение стрелками.
- **Что копировать** — галочки у баз SQL и папок; справа индикаторы, что найдено на компьютере.
- Внизу — **«Сделать резервную копию»** и **«Настройки»**.

## Первый запуск

Программа **сама** на этом компьютере:

- ищет **SQL Server** и список баз;
- при включённой опции в настройках — **файловые базы 1С** из списка пользователя (файл `ibases.v8i`).

Отметьте галочками, что войдёт в архив. В **настройках** можно указать сервер SQL, добавить или убрать папки, импортировать список из `.v8i`, выбрать сжатие ZIP.

## Настройки (три вкладки)

**SQL Server (1С)** — локальный или удалённый сервер, проверка подключения, список баз на сервере.

**Папки** — каталоги для копирования; для файловых баз 1С — метка «1С», опция «только файл .1CD», сжатие ZIP.

**Носитель и прочее** — куда сохранять архив:

| Вариант | Описание |
|:--------|:---------|
| **Только съёмный USB** | Флешка, автоопределение |
| **На любой диск** | Локальный или съёмный диск (буква вручную или авто) |
| **В папку на этом ПК** | Указанный каталог (с предупреждением о риске) |

Также: имя подпапки архивов (`Backups` по умолчанию), формат даты в имени папки, текст подсказки в окне программы, опция **запоминать отмеченные галочки** на следующий запуск.

## Каждый бэкап

1. Подключите носитель или проверьте путь в настройках.
2. Отметьте, что копировать — строка **«Нужно ~X · свободно Y»** покажет, хватит ли места.
3. Нажмите **«Сделать резервную копию»** и дождитесь окончания.

Если места мало, кнопка неактивна. Если место кончится **во время** копирования, архив прервётся, неполная папка удалится (если возможно).

Если папка с такой датой **уже есть**, программа **не перезапишет** её без вашего решения.

## Что лежит в архиве

```
Backups\2026-05-21\
  SQL\          ← файлы .bak (базы 1С)
  Folders\      ← скопированные папки (если выбраны)
  manifest.json ← что и когда сохранили
```

## Частые вопросы

**Нужен ли интернет?**  
Нет, всё локально.

**Заменяет штатный бэкап SQL?**  
Нет. Это **дополнительная** ручная копия, когда вам удобно.

**Можно без SQL, только папки?**  
Да.

**Где настройки программы?**  
`%ProgramData%\ITBackup\`
